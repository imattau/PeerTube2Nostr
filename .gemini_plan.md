# PeerTube2Nostr Evolution Plan

## Phase 1: Mitigating Threats (High Priority)
1. **Dependency Pinning:** Pin all versions in `requirements.txt` and `package.json` to prevent breaking changes.
2. **Relay Health Monitor:** 
    - Backend: Add a background task to ping relays and store latency/availability.
    - Frontend: Visual indicators for relay status (Online/Offline/Slow).
3. **Resource Optimization:** Refactor `AppManager` and `Runner` to use `asyncio` natively where possible to reduce thread overhead.

## Phase 2: Strengthening Weaknesses (Medium Priority)
1. **Security Layer:**
    - Add a simple API Key authentication for all `/api/*` routes.
    - Tighten CORS policy to only allow the specific frontend origin.
2. **Robust UI Data Entry:**
    - Replace browser `prompt()` with Vue 3 Modals.
    - Implement Regex-based validation for URLs (PeerTube, RSS, Relays).
    - Implement NSEC validation (check bech32 format).
3. **Database Resilience:** Optimize SQLite for concurrent access using a more robust connection management pattern.

## Phase 3: Exploiting Opportunities (Innovation)
1. **Real-time Streaming (WebSockets):** 
    - Use FastAPI WebSockets to stream logs directly to the Vue UI.
    - Push metric updates instead of polling every 5 seconds.
2. **Rich Media Integration:** 
    - Update `PeerTubeClient` to fetch thumbnail URLs.
    - Display thumbnails in the "Pending" and "Posted" lists on the UI.
3. **PWA Transformation:** 
    - Add `manifest.json` and a Service Worker to allow "Add to Home Screen" on mobile devices.
